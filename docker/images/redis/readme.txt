Логика очистки данных в редисе

У нас есть 3 группы данных:

1) Кеш хибера (сущности и запросы)

    а) Автоудаление неактивных записей по истечении 8 часов. Это делает сам хибер
        Он хранит время жизни в отдельных объектах в redis (не TTL)
        см. application.yml - redisson: entity: expiration:
    б) Автоудаление на серваке с maxmemory-policy allkeys-lfu (hcache.conf)
        Это автоудаление любых (не только имеющих TTL) наиболее редко используемых записей при исчерпании maxmemory

2) Кеш результатов методов

    Автоудаление на серваке с maxmemory-policy volatile-lru (commom.conf)
    Это автоудаление только имеющих TTL самых старых записей при исчерпании maxmemory
    TTL записей кеша методов: см. RedisConfig: RedisCacheManager...entryTtl(...)

3) Персистентные сущности @RedisHash (пример: LockInfo)

    В общем случае они живут вечно, но можно настроить им TTL (RedisHash...timeToLive)
    В этом случае они будут подвержены автоудалению при соответствующей настройке maxmemory-policy
